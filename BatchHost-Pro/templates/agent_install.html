Installation Guide
Step-by-step instructions for setting up the agent as a persistent background service
1
Prerequisites
Ensure Python 3.8+ is installed and available on the system PATH. Run the following to verify: 
python --version
# Expected: Python 3.x.x
pip install psutil requests
ℹ The agent can also run without pip packages using the built-in batchhost-agent-service.exe wrapper — which bundles its own runtime.
2
Extract & Place Files
Download the Windows ZIP, then extract it to a permanent location accessible by the SYSTEM account. The recommended path is: 
# Recommended installation path
C:\BatchHost-Pro-Agent\

# Create the directory (run as Administrator)
mkdir C:\BatchHost-Pro-Agent

# Copy contents of batchhost-agent-windows\ into it
xcopy /E /I batchhost-agent-windows C:\BatchHost-Pro-Agent
3
Configure the Server URL
Open batchhost-pro_agent.bat in Notepad and verify the server URL is set correctly: 
:: Line ~62 in batchhost-pro_agent.bat
set SERVER_URL=https://172.100.30.191:5000
⚠ If you have configured BatchHost-Pro on a different IP or port, update this value accordingly before starting the agent.
4
Install as a Windows Service (recommended for servers)
Use the included batchhost-agent-service.exe (WinSW wrapper) to register the agent as a proper Windows Service that auto-starts with the OS and restarts on failure. 
# Open PowerShell as Administrator in the agent folder:
cd C:\BatchHost-Pro-Agent

# Install the service:
.\batchhost-agent-service.exe install

# Start the service:
.\batchhost-agent-service.exe start

# Verify it is running:
Get-Service -Name "BatchHost-Pro_Agent"

# To stop / uninstall later:
.\batchhost-agent-service.exe stop
.\batchhost-agent-service.exe uninstall
✓ The service is configured to restart automatically on failure (10s then 30s delays) and starts with Windows boot.
5
Verify Registration
After starting, the agent registers automatically with https://172.100.30.191:5000. Within 30 seconds you should see it appear as Online in the BatchHost-Pro Agents page. 
# Check agent logs (from cmd/PowerShell):
type "%ProgramData%\BatchHost-Pro\Agent\logs\agent.log"

# Or if using the service:
type "C:\BatchHost-Pro-Agent\logs\BatchHost-Pro_Agent.out.log"
1
Prerequisites
Ensure Python 3.8+, curl, and bash are installed. 
# RHEL / CentOS / Rocky Linux
sudo dnf install -y python3 python3-pip curl

# Ubuntu / Debian
sudo apt-get install -y python3 python3-pip curl

# Verify
python3 --version && curl --version | head -1
# Install required Python packages
pip3 install psutil requests
2
Extract & Place Files
Upload the Linux ZIP to your server, then extract it to the recommended installation directory: 
# Create install directory
sudo mkdir -p /opt/batchhost-pro-agent

# Extract the ZIP (if unzip is available):
sudo unzip batchhost-agent-linux.zip -d /opt/

# Or using Python:
sudo python3 -c "import zipfile; zipfile.ZipFile('batchhost-agent-linux.zip').extractall('/opt/')"

# Set correct permissions
sudo chmod +x /opt/batchhost-pro-agent/batchhost-pro_agent.sh
sudo chmod +x /opt/batchhost-pro-agent/agent_runtime.py
3
Configure the Server URL
Open the agent script and verify the SERVER_URL points to your BatchHost-Pro server: 
# Edit with nano or vi:
sudo nano /opt/batchhost-pro-agent/batchhost-pro_agent.sh

# Line ~13 — verify this value:
SERVER_URL="https://172.100.30.191:5000"
HEARTBEAT_INTERVAL=10  # seconds between heartbeats
4
Option A — Run Manually (for testing)
Start the agent in the foreground to verify connectivity before setting up the service: 
cd /opt/batchhost-pro-agent
bash batchhost-pro_agent.sh

# Or run agent_runtime.py directly:
python3 agent_runtime.py daemon
4B
Option B — Install as a systemd Service (recommended)
Use the included batchhost-pro-agent.service unit file to register the agent as a systemd service that starts automatically on boot: 
# Copy the unit file to systemd
sudo cp /opt/batchhost-pro-agent/batchhost-pro-agent.service \
    /etc/systemd/system/batchhost-pro-agent.service

# Reload systemd and enable the service
sudo systemctl daemon-reload
sudo systemctl enable batchhost-pro-agent
sudo systemctl start  batchhost-pro-agent

# Check status
sudo systemctl status batchhost-pro-agent
✓ The service is configured with Restart=always and RestartSec=10, so it automatically recovers from crashes.
5
Firewall — Allow Outbound to BatchHost-Pro
The agent makes outbound HTTPS connections to port 5000 on the BatchHost-Pro server. Ensure your firewall allows this: 
# If using firewalld (RHEL/CentOS):
sudo firewall-cmd --permanent --add-rich-rule=\
  'rule family=ipv4 destination address=172.100.30.191/32 port port=5000 protocol=tcp accept'
sudo firewall-cmd --reload

# If using ufw (Ubuntu/Debian):
sudo ufw allow out to 172.100.30.191 port 5000 proto tcp
6
Verify Registration
After starting, the agent registers automatically with https://172.100.30.191:5000. Within 30 seconds it should appear as Online in the BatchHost-Pro Agents page. 
# View live service logs:
sudo journalctl -u batchhost-pro-agent -f

# Or check the log file directly:
tail -f ~/.local/state/batchhost-pro/agent/logs/agent_$(date +%Y-%m-%d).log

# Quick connectivity test (from the agent machine):
curl -k -s https://172.100.30.191:5000/api/dashboard/stats | python3 -m json.tool
⚠ The agent uses curl -k (skip SSL verification) by default because BatchHost-Pro uses a self-signed certificate. This is safe for internal network deployments.
7
Service Management Commands
Common commands for managing the agent service:
sudo systemctl start   batchhost-pro-agent  # Start
sudo systemctl stop    batchhost-pro-agent  # Stop
sudo systemctl restart batchhost-pro-agent  # Restart
sudo systemctl disable batchhost-pro-agent  # Remove from autostart
sudo systemctl status  batchhost-pro-agent  # Check status
